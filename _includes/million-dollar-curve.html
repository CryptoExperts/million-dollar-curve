Introducing the Million Dollar Curve

The Million Dollar Curve will be an alternative to curves P-256 and Curve 25519 to instantiate the ECDH cryptosystem, and will be produced in February 2016. This 128-bit secure elliptic curve is to be generated following the full methodology described in the technical paper and demonstrated in the tutorial to generate the Three Cents Curve. In particular, by committing publicly on the (deterministic) process and the list of draws before they actually happen, our methodology is designed so that nobody (even us!) can put a trap in the Million Dollar Curve.

Our objective is for the Million Dollar Curve to inspire confidence to the greatest number. In that sense, we want to integrate comments we receive in the methodology before committing on the final methodology, final source code and the lottery draws we will use.

You can direct any comment you have to us and we will gladly listen. Email us!

In what follows, we describe how we will generate the MDC201601 Curve (Million Dollar Curve of January 2016).

Our methodology can easily be adapted (and reproducible by anyone) for new curves with possibly different specifications (e.g. not an Edwards curve or aiming at 256 bits of security), and of course new lottery draws.

We will likely generate new Million Dollar Curves in the future on this website as needed.

Timeline to generate the MDC201601 Curve

  • Phase I: Before January 25th 2016 Current phase

    Receiving comments on our methodology

    In order for the Million Dollar Curve to win its spur, we greatly welcome any comment to improve the methodology we propose.

    Email us!
  • Phase II: January 27th 2016 Waiting

    Commitment on the design to generate MDC201601

    On January 27th 2016, we will present a full document (a text file) that describes the generation of Million Dollar Curve of January 2016 (named MDC201601), the list of security criteria we consider, and the acceptable set of parameters.

    The commit on the design (i.e. on the text file) will consist of:

    • A Let's Encrypt (U.S.) certificate for a subdomain of the form 
      MDC201601.SHA256.[xxx].cryptoexperts.com
      where [xxx] represents the SHA-256 sum of the aforementioned file split by dots every 16 characters.
    • A tweet on CryptoExperts' Twitter profile (U.S.).
    • The publication of the text file on CryptoExperts' Github million-dollar-curve directory (U.S.) and Bitbucket million-dollar-curve directory (Aus).
    • A video displaying the SHA-256 and SHA-512 of the text file in DailyMotion (FR).
    • A recording of Siri while reading every hexadecimal character of the SHA256 and of the SHA512 of the text file published on SoundCloud (DE).
    • An image of the handwritten SHA-256 and SHA-512 of the text file on Flickr (U.S.).
    • A note in the Cryptology ePrint Archive of the technical paper containing the SHA-256 of the text file
    • A hash from the hash chain service TrueTimeStamp.
    • Timestamps from the digital notaries Virtual-Notary and StartSSL.
    • Timestamp on the Bitcoin blockchain from BTProof.
    and the full list will be made public on this website.

  • Phase III: January 29th 2016 Waiting

    Commitment on the list of draws to consider in February 2016

    On January 29th 2016, we commit on a text file containing a list of future draws (starting on February 1st, 2016), a clear pointer to the aforementioned design (Phase II), and a source code that deterministically computes the seed from the listed draws.

    The commit on the resulting text file will consist of the same means as for the commit on the design (Phase II).

  • Phase IV: February 1st - February 13rd 2016 Waiting

    Lottery Draws

    During this period, the draws on which we committed on in Phase III should take place, and the full list of result will be gathered.

  • Phase V: February 14th 2016 Waiting

    Curve generation

    On February 14th 2016, all the draws committed on will have (or will not have) took place, and the full list of results will be made public.

    Using the source code committed on in Phase III, we will generate the seed from the full list of results.

    Using the source code committed on in Phase II and the aforementioned seed, we will generate the MDC201601 curve, and update this website accordingly.
